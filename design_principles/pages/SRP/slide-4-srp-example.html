Single Responsibility Principle (SRP)

How - Fixed Implementation
public class OrderValidator
{
    public void Validate() 
    { 
        Console.WriteLine("Validating order..."); 
    }
}

public class OrderRepository
{
    public void Save() 
    { 
        Console.WriteLine("Saving order..."); 
    }
}
public class PaymentService
{
    public void Charge() 
    { 
        Console.WriteLine("Charging payment..."); 
    }
}

public class EmailService
{
    public void SendConfirmation() 
    { 
        Console.WriteLine("Sending confirmation email..."); 
    }
}

Individual Service Classes:

✓ Each class has a single, well-defined responsibility
✓ Easy to test each service independently
✓ Services can be reused in other parts of the system
✓ Changes to one service don't affect others
